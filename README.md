# Dirt

Home grow monitoring and tracking system. Dirt runs local hardware automation for the grow box, a local web/API surface, a Railway-hosted control plane for away-from-home inspection and PTZ-only commands, an MCP server for Claude Desktop, and an agent-maintained wiki that synthesizes daily photos + sensor readings + chat notes into a structured knowledge base.

## Layout

- `apps/hwd/` — hardware daemon (ESP32 ingest, humidifier loop, capture, local automation) on :8000
- `apps/web/` — local UI + JSON API + MCP mount on :8001
- `apps/control-plane/` — hosted FastAPI control-plane API deployed to Railway service `control-plane-api`
- `apps/gateway/` — outbound-only local gateway that syncs local state/assets to cloud and polls PTZ-only command intent
- `apps/shared/` — models, db, services shared across apps
- `apps/mcp/` — MCP server mounted inside `dirt-web`
- `apps/voice/` — Claudia wake-word → Pipecat voice channel
- `web-ui/` — Vite + React + TanStack Router frontend for both local `dirt-web` and hosted Railway `web-ui`
- `cloud/` — Atlas-managed cloud schema and migrations for the hosted control plane
- `firmware/` — ESP32 firmware (fan controller, tent SHT45, reservoir and plant nodes)
- `wiki/` — agent-maintained grow knowledge base

## Runtime Surfaces

- **Local hardware plane:** `dirt-hwd` owns hardware-adjacent loops: sensor ingest, captures, humidifier control, fan/tent node integration, and local automation. It is not exposed publicly.
- **Local web plane:** `dirt-web` serves the local SPA/API/MCP surface on the LAN. It can still inspect and control local hardware through local services.
- **Hosted control plane:** Railway hosts `control-plane-api` and `web-ui`. The hosted UI shows the synchronized projection of `homebox/main` and `homebox/breeding`, recent private photos through signed URLs, freshness/offline state, and a narrow PTZ-only command surface. It does not talk directly to local hardware.
- **Local gateway:** `dirt-gateway` is a separate systemd service. It reads local Postgres, uploads selected state/assets to Railway, polls cloud command intent, validates scope locally, executes only allowed PTZ moves through local services, and reports results back to cloud.
